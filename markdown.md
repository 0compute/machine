# Markdown

- Embedded code blocks MUST include a language label.
- Embedded code MUST be fully validated, linted, and formatted.
- Embedded code MUST be syntactically valid and adhere to all language-specific
  standards defined in this document.
- Lists MUST use numbered (1., 1., 1.) or bullet (-) format.
- Nested lists MUST use indentation consistent with their parent list style.
- Relative links MUST use `./` prefix.
- Angle brackets MUST be escaped in prose; they MAY be used inside fenced code
  blocks when required by language syntax.
- List items and/or paragraphs with bold terms at start MUST NOT be used in
  place of headers. Doc MUST be semantically structured.
