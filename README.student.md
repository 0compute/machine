## Section 2: Human Expression as Code — Logic Chain

The analogy is precise, not decorative. Human speech is software. Here is why:

**Source Code = raw signal.** Energy, gesture, speech — the data before it enters the listener's runtime. It exists independently of whether it is received or interpreted correctly.

**Runtime = the mental imagery and emotion triggered in the listener.** This is the execution environment. Critical insight: the same source code runs differently in different runtimes. Two people hear the same words and construct different mental images. This is the root cause of Babylonian signal loss.

**Bloat = unnecessary words = wasted CPU cycles.** High-performance language is dense and direct. Every word that does not advance meaning degrades throughput.

**Technical Debt = filler ("um", "ah") and ambiguous phrasing = latent bugs.** Left unaddressed, they compound. A vague phrase today becomes a misunderstanding tomorrow. Refactoring (editing, rephrasing) is how you service the debt.

### Language Coverage and the Fidelity Threshold

The [Language Register](./languages.md) tracks which languages are Sovereign-capable and which are Perimeter-level (prone to signal loss). Fidelity is measured by a similarity score:

$$S < 0.88$$

Any translation scoring below 0.88 (88% semantic similarity) fails the Overstand Audit and is rejected. The threshold is binary: pass or fail. There is no "good enough" in Machine³.
